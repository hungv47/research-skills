# Hypothesis Guide

Adapted from the Watanabe framework and initiative hypothesis patterns.

## The If/Then/Because Structure

```
If [this cause is true]
Then [we'd expect to see this evidence]
Because [this mechanism explains it]
```

The **Because** is most important and most often skipped. Without it, you can't learn from failures.

## Templates by Problem Type

### Acquisition Problem
```
If [channel/source] quality declined
Then [conversion from that source] dropped while others stayed flat
Because [specific change: targeting, algorithm, competition]
```

### Retention Problem
```
If [user segment] is churning faster
Then [churn rate in segment] increased while others are stable
Because [trigger: price change, feature removal, competitor launch]
```

### Activation Problem
```
If [onboarding step] is failing
Then [drop-off at that step] increased
Because [specific friction: complexity, email deliverability, unclear value]
```

### Revenue Problem
```
If [pricing/packaging] is misaligned
Then [upgrade/expansion rate] declined for [segment]
Because [willingness-to-pay shifted due to: market change, feature perception, alternatives]
```

## Common Mistakes

1. **No "Because"** — "If churn increases, conversion drops." This is a tautology, not a hypothesis.
2. **Unfalsifiable** — "If the market shifts, things change." What specific evidence would prove this wrong?
3. **Multiple variables** — "If we changed pricing AND onboarding AND targeting..." You won't know which mattered.
4. **Vague evidence** — "Then metrics improve." Which metric? By how much? Compared to what baseline?
